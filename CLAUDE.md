# 작업 지침

이 저장소는 **여러 PC에서 이어서 진행하는 ERP/MES 도메인 학습 프로젝트**다. 대화 기록은 PC 간 공유되지 않으므로, 진행 상태는 저장소 파일로만 복원한다.

## 진행 현황 — 가장 먼저 읽을 것

- **`PROGRESS.md`** 가 진행 현황판이다. "다음 진도 실행" 요청을 받으면 **저장소 전체를 훑지 말고 `PROGRESS.md` 를 먼저 읽어** 현재 위치를 파악한 뒤, `git log -3` 으로만 가볍게 대조하고 바로 다음 단계로 진행한다.
- **하나의 진행 단계가 끝날 때마다** `PROGRESS.md` 의 "현재 위치"와 "Phase 상태표"를 갱신한다. (커밋·푸시는 사용자가 직접 하므로 자동으로 커밋하지 않는다.)
- **전체 학습이 끝나면** 사용자에게 `PROGRESS.md` 삭제 여부를 안내한다.
- 자세한 운영 규칙은 `PROGRESS.md` 상단 "운영 규칙" 절 참조.

## 학습 방식

- 커리큘럼: `ERP-STUDY-PLAN.md` (Phase 0~16, 7단계 사이클).
- Phase별 산출물 인덱스: `doc/README.md`.
- AI 역할: 도메인 브리핑 작성 / 모델링 제안 / 코드 구현 / 코드 워크스루 / 시연 가이드. 사용자는 코드를 직접 타이핑하지 않고 도메인 이해에 집중한다.
- 커밋·푸시는 사용자가 요청할 때만.
