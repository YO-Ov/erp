# 전체 시스템 조감도 — 현우전자 ERP + MES

> **이 글의 목적**: 개별 모듈 학습에 들어가기 전에 **"숲"**을 먼저 본다.
> 이 시스템이 현실에서 무슨 일을 하고, 조각들이 어떻게 맞물리는지 — 코드가 아니라 큰 그림.
> 이후 각 주제는 **①도메인(업무) 설명 → ②개발(구현) 설명** 2단계로 이어진다.

---

## 0. 한 문장으로

> **현우전자는 노트북을 "만들어서" "판다".** 회사의 모든 업무를 두 시스템이 나눠 처리한다 —
> 본사 사무실은 **ERP**, 공장 현장은 **MES**.

---

## 1. ERP 의 5개 모듈 (사무실에서 벌어지는 일)

ERP 는 부서별 모듈로 나뉜다. 약어가 많지만 **현실의 부서**로 생각하면 쉽다.

| 약어 | 풀이 | 현실의 누구 | 하는 일 |
| --- | --- | --- | --- |
| **SD** | Sales & Distribution | 영업팀 | 견적 → 수주 → 출하 → 세금계산서 (판매) |
| **MM** | Materials Management | 구매·자재팀 | 재고, 입고, 출고, 창고 |
| **FI** | Financial Accounting | 재무팀 | 회계 전표(복식부기), 채권/채무, 입출금 |
| **HR** | Human Resources | 인사팀 | 직원, 근태, 급여 |
| **PP** | Production Planning | 생산관리팀 | BOM(제품 레시피), 생산지시 |

---

## 2. ERP 의 핵심 — "모듈은 따로 놀지 않는다" ⭐

ERP 가 단순 CRUD 와 다른 결정적 이유. **거래 하나가 여러 부서를 동시에 건드린다.**

예: **노트북을 출하(SD)하면**
```
영업: 출하 확정
  ├─▶ 자재(MM): 창고 재고 자동 차감
  └─▶ 회계(FI): "매출원가" 전표 자동 생성
이어서 세금계산서 발행:
  └─▶ 회계(FI): "매출 / 부가세 / 매출채권" 전표 자동 생성
```

영업 담당자는 "출하" 버튼만 눌렀는데 **재고가 줄고 회계 장부가 자동으로 써진다.**
코드에서는 이것을 **이벤트(Spring Events)** 로 모듈을 느슨하게 연결해 구현했다.

---

## 3. 두 가지 큰 업무 흐름

**① OTC (Order-to-Cash, 돈 버는 흐름)**
```
견적 → 수주 → 출하(재고↓) → 세금계산서(매출↑) → 수금(채권 소멸)
```

**② 제조 흐름 (물건 만드는 흐름)**
```
BOM(레시피) → 생산지시 → [부품 소모 + 완제품 생산] → 재고↑ → 원가 계산
```

두 흐름은 **재고(MM)** 와 **회계(FI)** 에서 만난다. 그래서 MM·FI 가 ERP 의 중심축이다.

---

## 4. "실제 제조"는 공장에서 — MES

ERP 의 PP 는 **"계획"** 만 한다("노트북 10대 만들어라"). **실제 현장 작업**은 별도 시스템 **MES** 가 한다.

```
[ERP] 생산지시 ──① "이거 만들어"(동기 REST)──▶ [MES] 작업지시 받음
                                                  ↓ 현장: 시작 → 실적 → 품질검사
[ERP] 재고↑·원가 ◀──② "이만큼 만들었어"(비동기 Kafka)── [MES] 실적 발생
```

**왜 나눴나?** 계획(돈·사무)과 실행(현장)은 관심사가 다르고, 실무에서도 벤더가 다르다.
그리고 **"두 시스템이 데이터를 안전하게 주고받는 법"** 을 배우는 게 MES 파트의 진짜 목적이다.

---

## 5. 두 시스템을 잇는 두 개의 다리

| | 동기 REST | 비동기 Kafka |
| --- | --- | --- |
| 방향 | ERP → MES (지시) | MES → ERP (실적 통보) |
| 성격 | "이거 해" (즉시 응답 필요) | "이렇게 됐어" (안전 전달이 중요) |
| 안전장치 | **Circuit Breaker** (MES 죽어도 ERP 보호) | **Outbox 패턴**(DB저장+메시지발행 원자성) + **멱등성**(중복 무시) |

---

## 6. 세 층위로 기억하기

1. **ERP 한 시스템 안** — 거래 하나가 영업·재고·회계를 자동으로 엮는다 (이벤트·자동분개)
2. **제조** — 계획(ERP PP)과 실행(MES)의 분리
3. **두 시스템 사이** — 동기(REST)/비동기(Kafka)와 그 안전장치들
