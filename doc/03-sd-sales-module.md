# SD(영업, Sales & Distribution) 모듈

## "영업"이 뭐냐면

회사가 **물건이나 서비스를 고객에게 팔고, 전달하고, 돈을 받는 모든 과정**을 말합니다.

일상에서 "영업한다"고 하면 흔히 "고객 만나서 계약 따오는 일"만 떠올리지만, ERP에서 말하는 **영업(SD) 모듈**은 더 넓은 의미입니다.

## SD 모듈이 처리하는 업무 흐름

영업의 전체 사이클(이걸 **OTC, Order-to-Cash**라고 합니다)은 다음과 같습니다.

```
1. 고객 등록           (누구한테 팔 건지)
        ↓
2. 견적 (Quotation)    (얼마에 팔지 제안)
        ↓
3. 수주 (Sales Order)  (고객이 "살게요" 주문 확정)
        ↓
4. 출하 (Delivery)     (창고에서 물건 빼서 배송)
        ↓
5. 세금계산서 발행      (Invoice)
        ↓
6. 수금 (Payment)      (고객이 돈 입금)
```

이 전체 과정을 관리하는 것이 **SD 모듈**입니다.

## 구체적인 예시

**상황**: 삼성전자가 LG디스플레이에 패널 1,000개 주문

| 단계 | SD에서 하는 일 | 다른 모듈 연계 |
|------|---------------|----------------|
| 수주 | 주문서 입력 (고객, 품목, 수량, 단가) | - |
| 재고 확인 | 창고에 1,000개 있는지 확인 | **MM(자재)** 호출 |
| 생산 의뢰 | 부족하면 생산 요청 | **PP(생산)** 연계 |
| 출하 | 창고에서 1,000개 출고 처리 | **MM** 재고 차감 |
| 세금계산서 | 매출 1억원 인보이스 발행 | **FI(회계)** 매출 전표 자동 생성 |
| 수금 | 1억원 입금 확인 | **FI** 채권 소멸 |

→ SD는 **혼자 일하지 않고 MM, PP, FI와 계속 데이터를 주고받습니다**. 이것이 바로 ERP의 핵심입니다.

## SD가 관리하는 핵심 데이터

- **고객 마스터(Customer Master)**: 거래처 정보 (회사명, 주소, 신용 한도, 결제 조건)
- **가격 마스터(Pricing)**: 고객별/품목별 단가, 할인율
- **수주(Sales Order)**: 주문 내역
- **출하(Delivery)**: 배송 정보
- **인보이스(Billing)**: 청구서, 세금계산서

## "영업"의 다른 뜻과 구분

| 용어 | 의미 |
|------|------|
| **영업(Sales)** = SD 모듈 | 주문~수금까지의 거래 처리 |
| **영업이익** | 회계 용어, 매출에서 비용 뺀 이익 |
| **영업 사원(Salesperson)** | 고객 만나서 계약 따오는 사람 (CRM 영역) |
| **영업일(Business Day)** | 회사가 일하는 날 (월~금) |

ERP에서 "SD 모듈 = 영업"이라고 하면 **"수주부터 수금까지의 거래 처리 시스템"**을 의미합니다.

## 미니 ERP에서 SD를 만든다면?

가장 간단한 형태로:

```
[고객 테이블]    [상품 테이블]
      ↓              ↓
   [주문 테이블 (Sales Order)]
      ↓
   [주문 상세 테이블 (Order Item)]
      ↓
   [출하 테이블 (Delivery)]
      ↓
   [인보이스 테이블 (Invoice)]
```

이 정도만 만들어도 "고객이 주문 → 재고 차감 → 매출 발생" 흐름을 체험할 수 있습니다.
