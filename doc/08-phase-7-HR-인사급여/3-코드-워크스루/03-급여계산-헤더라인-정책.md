# 3/6. 급여대장 — 헤더-라인, 그리고 세율을 코드 한곳에 모은 정책

> 계약(급여 기준)과 근태(실제 근무)를 합치면 한 직원의 한 달 명세서가 나온다.
> `PayrollRun`(급여대장 = 헤더) ↔ `Payslip`(명세서 = 라인) 은 Phase 2 의 수주 헤더-라인과 같은 골격이고,
> 계산 규칙은 `PayrollPolicy` 한 클래스에 상수로 모았다.
> 이 글은 **기본급+연장수당 → gross → 공제 → net** 의 흐름이 코드로 어떻게 떨어지는지를 본다.

대상 파일:

```
hwlee-erp/src/main/java/com/hwlee/erp/hr/payroll/
├─ PayrollRun.java        헤더 (월 1건, 합계 캐시, 상태머신)
├─ Payslip.java           라인 (직원별 명세, 파생 계산)
├─ PayrollPolicy.java     세율/배율 상수 + 계산 헬퍼
├─ PayrollStatus.java     DRAFT / CONFIRMED / PAID
├─ PayrollService.java    계약+근태 → 명세서 생성
└─ PayrollController.java /api/payroll-runs (HR/ADMIN)
```

## 🔥 헤더-라인 — "한 달 대장 + 직원별 명세"

```
payroll_run (헤더)              "2026년 5월 급여대장"
├─ period       2026-05  (UNIQUE — 한 달에 대장 하나)
├─ status       DRAFT → CONFIRMED → PAID
├─ total_gross / total_deduction / total_net   ← payslip 집계 캐시
└─ payslips[]
     ├─ Payslip  김영업: 기본급 + 연장수당 → gross → 공제 → net
     ├─ Payslip  이재무: ...
     └─ Payslip  ...
```

Phase 2(수주 헤더-라인), Phase 5(전표 헤더-라인)에서 본 그 패턴이다. `period` 에 UNIQUE 를 걸어 **한 달에 급여대장은 하나** 임을 DB가 보장한다.

합계 3컬럼(`total_gross/deduction/net`)은 payslip 에서 더한 **캐시**다 — 대장 목록을 볼 때 라인을 다 펼치지 않고 합계만 보여주기 위함. 라인을 추가할 때마다 즉시 재계산한다:

```java
public Payslip addPayslip(Employee employee, BigDecimal basePay, BigDecimal overtimePay,
                          BigDecimal incomeTax, BigDecimal insuranceEmployee, BigDecimal insuranceCompany) {
    if (status != PayrollStatus.DRAFT)
        throw new IllegalStateException("DRAFT 급여대장에만 명세를 추가할 수 있습니다. 현재: " + status);
    Payslip slip = new Payslip(this, employee, basePay, overtimePay,
            incomeTax, insuranceEmployee, insuranceCompany);
    payslips.add(slip);
    recalculateTotals();   // ← 합계 캐시 즉시 갱신
    return slip;
}
```

## 🔥 세율을 코드 한곳에 — `PayrollPolicy`

급여 계산의 "마법 숫자"(6%, 4.5%, 1.5배)를 서비스 곳곳에 흩뿌리면 나중에 못 찾는다. 한 클래스에 상수로 모아 **한눈에 보이게** 했다:

```java
public final class PayrollPolicy {

    /** 연장근로 가산 배율 — 통상임금의 1.5배(근로기준법 가산율 50%). */
    public static final BigDecimal OVERTIME_MULTIPLIER = new BigDecimal("1.5");

    /** 소득세 간이 요율 6%. */
    public static final BigDecimal INCOME_TAX_RATE = new BigDecimal("0.06");

    /** 4대보험 직원 부담 요율 4.5% (월급에서 공제). */
    public static final BigDecimal INSURANCE_EMPLOYEE_RATE = new BigDecimal("0.045");

    /** 4대보험 회사 부담 요율 4.5% (공제 아님 — 회사가 추가로 부담 = 법정복리비). */
    public static final BigDecimal INSURANCE_COMPANY_RATE = new BigDecimal("0.045");
```

계산 헬퍼도 같은 클래스에 — 금액은 모두 원 단위 반올림:

```java
/** 연장수당 = 시급 × 1.5 × (연장근로 분 / 60). */
public static BigDecimal overtimePay(BigDecimal hourlyWage, int overtimeMinutes) {
    if (overtimeMinutes <= 0) return BigDecimal.ZERO.setScale(0);
    return hourlyWage
            .multiply(OVERTIME_MULTIPLIER)
            .multiply(BigDecimal.valueOf(overtimeMinutes))
            .divide(MINUTES_PER_HOUR, 0, RoundingMode.HALF_UP);
}

public static BigDecimal incomeTax(BigDecimal grossPay) {
    return grossPay.multiply(INCOME_TAX_RATE).setScale(0, RoundingMode.HALF_UP);
}
// insuranceEmployee / insuranceCompany 도 동일 (× 4.5%)
```

> 💡 **이 값들은 학습용 고정값이다.** 실제 4대보험은 항목별(국민연금·건강·고용·산재) 요율이 다르고 상한이 있다. 하지만 Phase 7 의 목적은 "근태→급여→공제→실수령→회계"의 **구조**를 보는 것 — 그래서 요율 한 묶음으로 단순화하고, 대신 그 값이 어디서 오는지 한눈에 보이도록 상수로 분리했다. 실무화는 이 상수를 "요율 테이블 조회"로 바꾸면 된다.

## 🔥 명세서 한 장의 계산 — `Payslip` 이 net 을 도출한다

라인 객체는 받은 값에서 gross / total_deduction / net 을 **스스로 도출**한다. 서비스가 net 을 계산해서 넘기는 게 아니다:

```java
Payslip(PayrollRun run, Employee employee, BigDecimal basePay, BigDecimal overtimePay,
        BigDecimal incomeTax, BigDecimal insuranceEmployee, BigDecimal insuranceCompany) {
    this.basePay = basePay;
    this.overtimePay = overtimePay;
    this.grossPay = basePay.add(overtimePay);                       // gross = 기본급 + 연장수당
    this.incomeTax = incomeTax;
    this.insuranceEmployee = insuranceEmployee;
    this.insuranceCompany = insuranceCompany;
    this.totalDeduction = incomeTax.add(insuranceEmployee);         // 공제 = 소득세 + 직원분 보험
    this.netPay = this.grossPay.subtract(this.totalDeduction);      // 실수령 = gross - 공제
}
```

**여기서 가장 중요한 한 줄** — `insurance_company`(회사분)가 `totalDeduction` 에 **들어가지 않는다**:

> 회사부담 4대보험은 **직원 월급에서 떼는 게 아니다.** 회사가 직원 한 명당 별도로 더 내는 비용이다. 그래서 실수령(net) 계산엔 안 들어가고, 회계에선 `법정복리비`(비용)로 잡힌다(다음 글). "회사가 직원 한 명에 쓰는 진짜 총비용 = gross + insurance_company".

이 비대칭 — **net = gross − 소득세 − 직원분보험** — 이 다음 글의 차대 균형이 **항상** 맞는 수학적 이유가 된다.

## 🔥 계약 + 근태 → 명세서 (PayrollService)

이제 조각이 모인다. 대상 월의 유효 계약 전부를 돌면서, 각 직원의 그 달 연장근로를 집계하고, 정책으로 공제까지 떨어뜨려 명세서를 만든다:

```java
@Transactional
public PayrollRunResponse createDraft(PayrollRunCreateRequest req) {
    YearMonth ym = YearMonth.parse(req.period());                 // "2026-05" → YearMonth
    if (repository.existsByPeriod(req.period()))
        throw new IllegalArgumentException("이미 해당 월의 급여대장이 있습니다. period=" + req.period());
    LocalDate firstDay = ym.atDay(1);
    LocalDate lastDay = ym.atEndOfMonth();

    String number = numberGenerator.nextPayrollNumber(ym);        // PR-202605-001
    PayrollRun run = PayrollRun.draft(number, req.period(), LocalDate.now(clock));

    List<EmploymentContract> contracts = contractRepository.findEffectiveOn(lastDay);   // 1글에서 본 쿼리
    for (EmploymentContract contract : contracts) {
        Employee employee = contract.getEmployee();
        int overtimeMinutes = attendanceRepository.sumOvertimeMinutes(employee.getId(), firstDay, lastDay); // 2글

        BigDecimal basePay = contract.getBaseSalary();                                   // 만근 가정
        BigDecimal overtimePay = PayrollPolicy.overtimePay(contract.hourlyWage(), overtimeMinutes);
        BigDecimal grossPay = basePay.add(overtimePay);
        BigDecimal incomeTax = PayrollPolicy.incomeTax(grossPay);
        BigDecimal insuranceEmployee = PayrollPolicy.insuranceEmployee(grossPay);
        BigDecimal insuranceCompany = PayrollPolicy.insuranceCompany(grossPay);

        run.addPayslip(employee, basePay, overtimePay, incomeTax, insuranceEmployee, insuranceCompany);
    }
    return mapper.toResponse(repository.save(run));
}
```

**만근 가정**: `basePay = 계약 기본급 전액`. 결근 일할 차감은 이번 범위 밖(복잡도 대비 학습가치 낮음). 단 **연장수당은 근태 기반으로 실제 계산** — "근태가 급여에 반영된다"는 핵심은 연장수당으로 체득한다.

예시 (기본급 3,000,000 / 소정 200h / 연장 120분):
```
시급        = 3,000,000 / 200            = 15,000
연장수당    = 15,000 × 1.5 × (120/60)    =    45,000
gross       = 3,000,000 + 45,000         = 3,045,000
소득세      = 3,045,000 × 6%             =   182,700
직원분 보험 = 3,045,000 × 4.5%           =   137,025
회사분 보험 = 3,045,000 × 4.5%           =   137,025   ← 공제 아님
공제 합     = 182,700 + 137,025          =   319,725
net(실수령) = 3,045,000 - 319,725        = 2,725,275
```

## 정리

| 개념 | 코드 |
| --- | --- |
| 월 1건 대장 | `payroll_run.period` UNIQUE + 합계 캐시 |
| 세율 한곳 | `PayrollPolicy` 상수 + 계산 헬퍼 |
| net 도출은 라인 책임 | `Payslip` 생성자가 gross/공제/net 계산 |
| 회사분 ≠ 공제 | `totalDeduction = 소득세 + 직원분` (회사분 제외) |
| 계약+근태 결합 | `findEffectiveOn` × `sumOvertimeMinutes` |

다음 글 ⭐: **인건비 자동 분개** — 이 명세서 합이 어떻게 복식부기 전표가 되고, 왜 차변 합 = 대변 합이 라운딩과 무관하게 항상 맞는가.
