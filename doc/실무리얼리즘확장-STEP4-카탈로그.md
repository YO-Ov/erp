### 기존 부품 10종 재분류 (PART → 세분)
- 15" LCD 패널·27" LCD 패널 → PART_DISPLAY
- 메인보드(노트북)·스케일러 보드·전원 보드 → PART_BOARD
- 16GB 메모리 → PART_MEMORY · 512GB SSD → PART_STORAGE · 배터리팩 → PART_BATTERY
- 모니터 스탠드·후면 케이스 → PART_CHASSIS

### BOM 방침
완제품 16종마다 BOM(5~8라인). 예시:
- 노트북 = LCD + 노트북메인보드 + CPU모듈 + 메모리 + SSD + 배터리 + 노트북케이스 + 전원어댑터
- 모니터 = LCD + 스케일러보드 + 전원보드 + 스탠드 + 후면케이스
- 데스크탑/WS = 데탑메인보드 + CPU(+GPU) + 메모리 + SSD + 데탑케이스 + 전원보드
- 키보드 = 키캡세트 + (기계식: 스위치) + 케이블 / 마우스 = 광센서 + 하우징 + 케이블
- 공용 부품(메모리/SSD/전원/CPU)이 여러 BOM에 동시에 등장.

- **유형 분포(파레토)**: 상위 ~20개 대형 고객이 매출의 다수. 유형 = 총판/리셀러·SI/시스템·제조/대기업구매·공공/교육.

## 10. 정합성 & 채번 메모 (생성 시 반드시)

1. **개시재고 ↔ 개시분개 일치**: 부품/완제품 기초재고 총액 = 개시 분개(차)재고 / 대)이익잉여금) 금액. (V35/V45 패턴 확장)
2. **BOM JOIN**: 기존 V35/V45는 부품 **이름(name)** 으로 JOIN → 신규 부품명 **유일**하게(중복 금지).
3. **채번 시퀀스**: `code_sequence` 의 ITEM/CUST/EMP/VEND next_number 를 시드 끝 번호+1 로 갱신.
4. **FK**: item.category → item_category.code (STEP2), warehouse.factory_id → factory.id (STEP3) 모두 충족하도록 카테고리/공장 먼저 INSERT.
5. 100건+ 규모는 **생성 스크립트로 SQL 산출** 후 Flyway 마이그레이션으로 편입(공유 RDS 1회 적용).

## 11. 향후 기능(STEP 4 범위 밖) — 전자결재(전사 횡단)

결재는 견적만이 아니라 **구매발주·지급·전표·근태/휴가·생산지시 등 여러 문서 공통**이다(hwlee님 지적).
→ 별도 설계 문서로 분리: **`doc/실무리얼리즘확장-전자결재-설계.md`** 참고.
- 권장: 어떤 문서든 붙는 **범용 결재 엔진**(approval_request/step/rule + 알림 + 문서 콜백).
- STEP 4에서 깔아둔 **부서별 담당+팀장 로그인**이 상신자/결재자가 됨.
- 진행 범위·타이밍은 미정(데이터 STEP 4~5 후 별도 단계 권장).
