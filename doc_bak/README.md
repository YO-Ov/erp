# ERP 학습 문서

ERP(전사적 자원 관리) 학습을 위한 도메인 정리 + Phase별 학습 산출물 모음.

## 폴더 구조

```
doc/
├─ 00-도메인-개론/             ← Phase 시작 전 ERP 도메인 워밍업
├─ 01-phase-0-환경-구축/       ← Phase 0 학습 사이클의 산출물 (브리핑/설계/워크스루/시연)
├─ 02-phase-1-마스터-데이터/   ← Phase 1 학습 사이클 (완료)
├─ 03-phase-2-SD-영업/         ← Phase 2 학습 사이클 (완료)
├─ 04-phase-3-MM-재고/         ← Phase 3 학습 사이클 (완료)
├─ 05-phase-4-SD-MM-연계/      ← Phase 4 학습 사이클 (완료)
├─ 06-phase-5-FI-회계/         ← Phase 5 학습 사이클 (완료)
├─ 07-phase-6-인증인가-감사로그/ ← Phase 6 학습 사이클 (완료)
├─ 08-phase-7-HR-인사급여/     ← Phase 7 학습 사이클 (완료)
├─ 09-phase-8-PP-생산/         ← Phase 8 학습 사이클 (구현 완료 — 워크스루/시연 남음)
├─ 10-phase-11-MES-셋업/       ← Phase 11 사전 브리핑 (시스템 구성도·기술 개요)
└─ 11-phase-16-통합-운영/      ← Phase 16 통합 아키텍처·E2E·운영 관점
```

명명 규칙:
- **부모 폴더**: `00-`, `01-`, `02-` ... 커리큘럼 순서. 도메인 개론은 `00`, Phase 0은 `01`, Phase 1은 `02`, ... 식.
- **자식 파일/폴더**: `0-`, `1-`, `2-` ... 한 Phase 안에서의 7단계 사이클 순서 (한 자리 숫자).
- **손주 파일** (코드 워크스루 등): `01-`, `02-` ... 묶음 안에서의 일련번호 (두 자리).

부모/자식의 숫자 형식을 일부러 다르게 해서 어느 층의 번호인지 한눈에 구분되도록 함.
